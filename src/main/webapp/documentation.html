Inheritance

Documentation

        Our inheritance strategy pretty much followed the model that was handed out for the assignment.

        Both the Person- and Company-entitys extends InfoEntity. 
        [Side note: Company class is not really used for anything in our project].

        We allowed Person to have multiple hobbies and the same hobby to be assigned to multiple persons, 
        so we used a “ManyToMany”-relation". 
        [Note: This originally resulted in a lot of critical and crashing errors, 
        which fixed with the help of Nikki, by adding “transient”to the variable declaration.

        A “OneToMany” relation was used between InfoEntity and Phone, to allow multiple Phones to be assigned to the same InfoEntity.

        A “ManyToOne” relation was used between InfoEntity and Address, so that we can have multiple InfoEnty’s and One Address assigned to them.  
        
        A “ManyToOne” relation between Address and CityInfo, so that CityInfo can have multiple Address and that a Address can only be assigned to one CityInfo.

Testing the System

How to Test the System
        
        REST Assured-test
        Before the Rest Assured Tests can run, the database must be populated with data. 
        To do this, make sure that the project has a persistence unit attached to the entities. 
        Then run the class “DatabaseInit” that is in the utility-package. 
        Make sure that data has been created by running the project, or using a mysql-viewer client. 
        Now that these steps have been completed, the Rest Assured Test can now be run. 
        This test class is located under the test packages in the rest package. 
        Each test method has been commented in the code.

        How to use the System
        Using the system is easy and intuitive. 
        The navigation bar at the top will give the option to view the project, open a link to GitHub, open a link to the Java Documentation 
        and the documentation of the project. 
        When viewing the project, two buttons at the top will let the user update the table, or create a new person. 
        The table will list all the persons on the database, and each user will have two options associated to it – delete and edit.
